# ai-pm-feedback-collector — PM 碎片信息收集与洞察整理

将来自客户会议、微信群、客服工单、竞品分析、用户访谈等多渠道的碎片化信息，整理为结构化的需求洞察文档。

## 定位

处于 PM 工作流的**上游**——在需求正式成型之前：

```
碎片信息（本工具）→ 结构化洞察 → PRD / 用户故事 / Issue 拆解
```

不生成 Issue（用 `/req-to-issues`），不写 PRD（用 `purvar-prd`）。

## 核心能力

- **主题聚类**：按业务主题/模块聚类，而非简单的 Bug/优化分类
- **JTBD 提取**：从每条反馈提取用户真实场景（When/I want to/So I can）
- **问题与方案分离**：区分用户的痛点（Opportunity）和用户建议的方案（Solution）
- **Opportunity Score**：基于 Dan Olsen 模型评分（重要性 × (1-满意度)）
- **情感标注**：标注反馈的正面/中性/负面情感倾向
- **下游衔接**：根据洞察结果推荐下一步行动和对应工具

## 使用方式

```
整理客户反馈
整理反馈
碎片信息整理
收集需求
```

然后粘贴来自任意渠道的碎片信息即可。

## 输出

`YYYY-MM-DD-反馈洞察-{主题}.md`，包含：
1. 信息来源摘要
2. 主题聚类（核心问题 + JTBD + 原始引用 + 情感 + 影响面 + Score）
3. 优先级排序表
4. 待验证假设清单
5. 下游衔接建议
