# PRD 澄清扫描分类

对已有功能迭代型PRD进行结构化扫描，识别模糊点和缺失项。共7个扫描维度，每个维度包含具体检查项。

## 1. 变更概要

检查业务诉求和方案概述是否清晰完整。

- [ ] 业务诉求是否一句话说清了痛点或目标
- [ ] 方案概述是否一句话说清了系统做什么
- [ ] 两者是否回答了"为什么做"和"做什么"
- [ ] 变更概要与后续功能说明是否一致（没有概要提到但功能说明没展开的内容）

**常见问题**：
- 业务诉求过于笼统（如"优化管理"）
- 方案概述包含了实现细节
- 概要和功能说明范围不一致

## 2. 影响范围

检查影响范围表格是否完整准确。

- [ ] 所有在功能说明中出现的页面/模块是否都在影响范围表中列出
- [ ] 变更类型是否准确（新增/修改/联动影响三选一）
- [ ] 是否遗漏了"联动影响"的模块（本次不直接改动但数据来源或行为会受影响的）
- [ ] 每行的"说明"是否一句话概括了影响

**常见问题**：
- 只列了直接修改的页面，遗漏了联动影响的模块
- 变更类型标错（如新增页面标成了"修改"）
- 功能说明中出现了影响范围表中没有的页面

## 3. 信息项完整性

检查六列信息项表格是否完整准确。

- [ ] 每个功能说明子章节是否都有信息项清单
- [ ] 有特殊说明的信息项是否单独列出（新增、改名、有显隐条件、有校验规则、有默认值的）
- [ ] 无特殊说明的信息项是否合并到末尾一行
- [ ] "类型"列是否使用了业务控件名称（文本输入、下拉选择等），而非技术名称
- [ ] 有无过多的"【待确认】"标记（超过5个需要关注）
- [ ] 信息项命名是否与业务规则中的措辞一致
- [ ] 新增字段是否在备注中标注了"新增字段"
- [ ] 改名字段是否在备注中注明了原名

**常见问题**：
- 信息项表格缺失（只写了业务规则没列信息项）
- 类型列写了技术名称（如"input""select"）
- 校验规则列全写了"—"，但明显有校验需求的字段（如金额、日期范围）

## 4. 业务规则

检查业务规则是否清晰无歧义。

- [ ] 每条业务规则是否可测试（能判断对错）
- [ ] 条件逻辑是否穷举所有分支（如选了A显示X，选了B呢？）
- [ ] 数据来源是否明确（"系统自动带出"——从哪里带？根据什么条件？）
- [ ] 审批流程中的角色和节点是否明确
- [ ] 多条规则之间是否有矛盾或冲突
- [ ] 是否存在模糊措辞（"相关""适当""必要时"等）

**常见问题**：
- 条件逻辑只写了"是"的分支，没写"否"
- "系统自动带出"没说明数据来源
- 审批流程节点之间的驳回路径不清楚

## 5. 异常与边界

检查异常与边界场景的覆盖度。

- [ ] 是否覆盖了数据冲突场景（并发编辑、引用的数据被删除）
- [ ] 是否覆盖了权限边界（无权限用户看到/操作此功能时的表现）
- [ ] 是否覆盖了输入边界（超长文本、特殊字符、数值极值）
- [ ] 是否覆盖了状态边界（已提交/已审核的记录能否编辑/删除）
- [ ] 每个场景的"预期行为"是否具体（不是"给出提示"而是写明提示内容）
- [ ] 是否遗漏了关联删除场景（删除记录时，引用它的其他记录怎么处理）

**常见问题**：
- 异常与边界章节为空或只有1-2条
- 预期行为写得太模糊（如"系统给出错误提示"——提示什么？）
- 缺少状态边界场景（审批流程涉及的各状态下的操作限制）

## 6. 验收标准

检查验收标准是否可验证。

- [ ] 是否使用了 checkbox 格式（- [ ]）
- [ ] 每条验收标准是否具体可验证（不含"正常""合理"等模糊词）
- [ ] 是否覆盖了功能说明中的所有核心业务规则
- [ ] 是否包含了关键异常流程的验收项
- [ ] 验收标准与功能说明是否一致（没有遗漏也没有多出来的）
- [ ] 信息项的显隐条件、默认值是否有对应的验收项

**常见问题**：
- 验收标准用了模糊表述（如"列表页展示正常""数据准确"）
- 只覆盖了正向流程，缺少异常流程的验收
- 功能说明中写了条件逻辑但验收标准没有验证各分支

## 7. 开放问题

检查开放问题是否完整处理。

- [ ] 所有"【待确认】"标记是否都记录在开放问题表中
- [ ] 开放问题的描述是否足够具体（不是"XX待确认"而是写清具体问什么）
- [ ] 状态标注是否准确（待确认/已确认）
- [ ] "已确认"的问题是否填写了结论
- [ ] 是否存在功能说明中标注了"【待确认】"但开放问题表中没有的

**常见问题**：
- 功能说明中有"【待确认】"但开放问题表中没有对应记录
- 问题描述太笼统，不知道具体要确认什么
- 已确认的问题没有回写到功能说明中（开放问题表更新了但正文还是"【待确认】"）

## 扫描输出格式

对每个维度评估后，标记状态：

| 状态 | 含义 |
|------|------|
| ✅ 清晰 | 该维度信息完整，无需澄清 |
| ⚠️ 部分 | 有可改进的地方，但不阻塞 |
| ❌ 缺失 | 关键信息缺失，需要澄清 |

优先处理 ❌ 缺失 > ⚠️ 部分，在同一状态内按维度编号优先级排序。
