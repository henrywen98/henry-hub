# ssot-prompt-engineer

SSOT 驱动的提示工程方法论。从输入-输出样例对中提取转换规则，将其编纂为单一事实来源（SSOT），然后从 SSOT 派生出提示词和工具链。适用于逆向工程 SOP、构建可复用的提示系统、将隐性领域知识形式化为结构化生成规则。

## 使用场景

当用户提到以下内容时触发：

- "extract rules" / "提取规则"
- "create SSOT" / "创建 SSOT"
- "reverse engineer the process" / "逆向工程流程"
- "build prompt from examples" / "从示例构建提示词"
- "formalize the SOP" / "形式化 SOP"
- "make a prompt toolkit" / "制作提示工具包"
- "SSOT to prompt"
- 分析输入如何映射到输出并提取底层规则
- 从现有示例或模板创建提示工具包
- 构建规则与提示模板分离的可复用提示系统

### 五阶段流程

1. **收集 I/O 样例对**：收集领域中的具体输入-输出示例
2. **逆向工程转换规则**：分析 I/O 对以提取映射逻辑
3. **编写 SSOT 文档**：将规则结构化为独立、自包含的文档
4. **从 SSOT 派生提示词**：生成系统提示、用户提示模板
5. **构建工具链并验证**：创建辅助工具并验证完整流水线

## 安装

```bash
/plugins install ssot-prompt-engineer@henry-hub
```
