# doc-coauthoring

结构化文档协作写作工作流。引导用户通过三个阶段（上下文收集、迭代优化、读者测试）高效完成文档撰写，适用于技术规格、决策文档、提案等各类结构化内容。

## 使用场景

当用户提到以下内容时自动触发：

- 撰写文档："write a doc"、"draft a proposal"、"create a spec"、"write up"
- 特定文档类型："PRD"、"design doc"、"decision doc"、"RFC"
- 开始一项较大规模的写作任务

### 工作流三阶段

1. **上下文收集**：用户提供所有相关上下文，Claude 提出澄清问题
2. **优化与结构化**：逐节迭代构建内容，包括头脑风暴和编辑
3. **读者测试**：使用全新的 Claude 实例（无上下文）测试文档，发现盲点

## 安装

```bash
/plugins install doc-coauthoring@henry-hub
```

## 使用示例

```
用户：帮我写一个技术设计文档
Claude：我可以引导你通过结构化的协作写作流程来完成这个文档。流程分为三个阶段...

用户：我需要写一个 RFC
Claude：好的，让我们开始上下文收集阶段。请告诉我...

用户：帮我起草一个项目提案
Claude：我来引导你完成提案的撰写。首先，这个文档的目标受众是谁？
```
