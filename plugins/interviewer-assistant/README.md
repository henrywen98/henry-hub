# interviewer-assistant

> 站在**面试官**一侧的结构化工作流工具：面试前从简历生成针对性面试题并识别包装痕迹；面试后对照会议纪要评估候选人真实水平。

**Category:** HR
**Scope:** 适用于任何岗位（产品、技术、市场等）的结构化面试

---

## 包含两个 skill

| Skill | 触发时机 | 输入 | 输出文件 |
|---|---|---|---|
| `pre-interview` | 面试前 | 简历 PDF/MD（+ 岗位 JD 可选） | `面试问题_<候选人>.md` |
| `post-interview` | 面试后 | 简历 + 会议纪要 | `面试复盘_<候选人>.md` |

## 使用示例

**面试前：**
```
你：帮我准备面试，@候选人简历.pdf，岗位是 AI PM，30 分钟
→ 自动读 PDF 转 MD
→ 扫描简历包装痕迹（结构模板化、术语堆砌、数据无口径、跨段矛盾等）
→ 生成 15-20 题，按 4 段式分布（暖场/经历深挖/技术理解/软素质）
→ 每道题配追问点 + 面试官备忘
→ 输出到 面试问题_<候选人>.md
```

**面试后：**
```
你：复盘这次面试，@简历.md + @meetingnotes
→ 两份材料对照读
→ 按 4 维评估（真实性/技术深度/数据素养/判断力）
→ 每条评估锚定纪要原文
→ 输出到 面试复盘_<候选人>.md（不给录用结论）
```

## 设计理念

### 1. 不做拍板，做证据整理
`post-interview` 硬性禁止给"录用/不录用"结论。它的工作是把证据摆清楚，让判断变容易——决定权始终在面试官手里。

### 2. 对抗简历包装
`pre-interview` 内置 8 类红旗识别清单（结构模板化、术语堆砌、数据无口径、身份成就不匹配等），追问题要求候选人把"方法论话术"讲到"具体事件 + 数据归因 + 判断取舍"。

### 3. 公平性护栏
- **偏见屏蔽**：明确禁止学校背景、年龄、性别、口音、外貌等非能力信号影响评估
- **AI 纪要偏差警示**：元宝/飞书等 AI 生成纪要存在漏记/误记/润色三类偏差，引用主观措辞必须标注"纪要中的助手评价"
- **单一证据标注**：当一个维度的判断只靠一条证据，要显式标注"单一证据，权重有限"

### 4. 方法论可迭代
红旗识别清单、追问框架、评估维度都放在各 skill 的 `references/` 下，用着用着发现新模式可以增量追加，SKILL.md 保持精简。

## 隐私与数据处理

- 输出文件包含候选人姓名、公司名等敏感信息，**面试官有责任妥善保管**
- **不要**把复盘输出发送到候选人可见的渠道（邮件、IM、协作文档等）
- 所有示例/模板中的候选人姓名、公司名均已匿名化为"候选人 A"、"某公司"等

## 文件约定

- 默认输出到**简历/纪要所在目录**（通常是候选人专属文件夹），不自动写入 Obsidian 或其他位置
- 默认只出 Markdown 格式；需要 DOCX 时单独说一声，用 pandoc 转
