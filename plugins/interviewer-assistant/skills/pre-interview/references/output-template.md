写到 `面试问题_<候选人姓名>.md`。结构如下 —— **核心必问段**是 30 分钟实际跑完的 6-8 题，**备选弹药段**是候选人答得顺时才追的深挖池。

```markdown

**候选人：** <学校/背景一句话概括>

## 面试官备忘：关注点

| 维度 | 关注什么 |
|---|---|
| **真实性** | <基于简历红旗具体化，例如："简历结构模板化，追问时让候选人跳出模板讲具体例子"> |
| **技术深度** | <基于候选人背景具体化，例如："跨专业转 AI PM，验证 RAG 理解是否到落地层"> |
| **数据素养** | <基于简历数据具体化，例如："简历 18%/22%/35% 三个指标都没说基线，挨个追问"> |
| **判断力** | <提醒面试官关注候选人是否做过有难度的取舍决策> |
| **诚实度** | <把 2-3 个"在不利问题上是否愿意讲真话"的题号列出来——例如年限 / ToG 口径题、时间线 vs 技术事件题、工具边界题。看候选人遇到不利问题时，是坦率承认还是辩解包装> |
| **矛盾点** | <简历里的具体矛盾，例如："个人优势提到 X 公司但实习经历没有，可直接问"> |
```

## 写题时的关键要点

1. **主问题简短**（≤ 2 行），**追问角度用括号**在主问题后面跟着
2. **面试官备忘用引用块 `>`**，视觉上和面试题区分开
3. **具体**——追问点必须锚定简历里的具体内容（"8000 道题"、"18% 留存"），不要写泛泛的追问
4. **核心段编号连续**（1、2、3...），**弹药段用字母前缀**（A1/B1/C1/D1），面试官扫一眼就知道哪是必问哪是备选
5. **全局备忘表**放在文末，不是开头——让面试官在面试结束后可以用这个表格做评估

## 示例（虚构候选人，AI 产品经理岗位核心段深挖模式）

```markdown
### 某在线教育公司（最近一段，重点问）

3. 你提到推动产品从"搜题工具"转型为"个性化学习伙伴"，能具体说说这个转型的背景吗？是你提出的还是团队已有方向？你在其中具体负责哪部分？
4. 留存提升 18%、停留时长增加 20%——这些数据是整个产品的还是你负责的功能模块的？A/B 测试的对照组是什么样的？

> **面试官备忘：** 简历写"主导"，应届生的话这里是诚实度试金石。
```

对应的**弹药段**示例：

```markdown
A1. 你说用 Prompt 工程生成了 8000 多道拔高题，这个过程能展开讲讲吗？Prompt 是你写的吗？怎么评估生成题目的质量？
    - **触发：** 若 Q3 候选人把转型项目讲清楚了，再追这题拆技术执行细节

B1. "错题-RAG-知识图谱"联动机制——能用白话解释一下这个系统的数据流吗？错题数据怎么进来，RAG 怎么检索，最后怎么给学生推荐内容？
    - **触发：** 若候选人在技术落地题（Q5）给得出 chunk / embedding 细节时
```

注意：
- 每道题都引用简历里的具体表述（转型、8000 多道、18%）
- 把方法论问题转成"具体怎么做"的问题
- 同一道题里嵌套 2-3 个追问角度
- 弹药段每题都明确触发条件，面试官现场不用再判断要不要问
