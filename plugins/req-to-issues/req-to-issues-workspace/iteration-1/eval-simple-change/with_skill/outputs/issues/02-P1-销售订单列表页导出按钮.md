## 需求描述

- 做什么：在销售订单列表页的工具栏添加"导出 Excel"按钮，点击后将当前筛选条件下的所有订单数据下载为 Excel 文件
- 为谁做：需要导出订单数据的业务人员
- 解决什么问题：用户在列表页筛选出想要的订单后，缺少一键导出为 Excel 的入口

## 业务价值

- 对用户：一键导出，操作路径短，无需切换到其他页面或工具
- 对业务：降低数据获取门槛，让更多人能自助获取所需的订单数据
- 不做会怎样：即使后端提供了导出能力，用户也没有可视化的操作入口来使用

## 验收标准

- [ ] 列表页工具栏（搜索框和新建按钮旁）出现"导出 Excel"按钮
- [ ] 点击按钮后，自动将当前页面上生效的所有筛选条件（状态 Tab、客户筛选、日期范围、搜索关键词、待审批筛选）传递给导出接口
- [ ] 导出期间按钮显示加载状态（loading），防止用户重复点击
- [ ] 导出完成后，浏览器自动触发文件下载
- [ ] 导出失败时（如网络错误、数据量超限），显示明确的错误提示信息
- [ ] 无订单数据时（筛选结果为空），点击导出给出"当前筛选条件下无订单数据"的提示，不发起无意义的下载请求
- [ ] 按钮仅对有销售查看权限的用户显示

## 影响范围

- 涉及模块：销售订单列表页
- 涉及角色：所有拥有销售查看权限的用户
