# 销售订单导出 Excel 功能 -- Issue 拆解汇总

## 原始需求

> 销售订单列表页加一个导出 Excel 的按钮，导出当前筛选条件下的所有订单数据

## 需求解析

- **业务目标**：让用户能从销售订单列表页一键导出筛选后的订单数据为 Excel 文件，方便离线分析和汇报
- **涉及角色**：所有拥有销售查看权限的用户（销售人员、管理员）
- **核心功能点**：后端导出接口 + 前端导出按钮

## 模糊点识别与假设

在分析需求时识别出以下模糊点，已基于常见业务实践做出合理假设：

| # | 模糊点 | 假设 | 理由 |
|---|--------|------|------|
| 1 | "所有订单数据"的范围 — 仅列表摘要字段还是包含明细行？ | 导出列表级摘要字段（与列表页表格列一致） | 明细行导出会使 Excel 结构复杂化，属于更高级的需求 |
| 2 | "所有订单"是否忽略分页？ | 是，导出所有匹配记录，不受分页限制 | 导出的目的就是获取完整数据集 |
| 3 | 谁可以导出？ | 拥有销售查看权限的用户 | 能看即能导出，符合最小权限原则 |
| 4 | 列标题语言、日期格式、文件命名 | 中文列标题，日期 YYYY-MM-DD HH:mm，文件名含日期 | 系统面向中文用户 |
| 5 | 大数据量处理 | 上限 10,000 条，超出提示缩小范围 | 平衡性能和实用性 |
| 6 | Excel 生成方式 | 后端生成（openpyxl 已安装），返回二进制下载 | 比前端生成更可靠，适合大数据量 |
