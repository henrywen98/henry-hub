## 需求描述

- 做什么：客户在对账单查看页面可以上传银行转账凭证（水单），填写付款金额、付款日期、付款银行等信息，提交付款通知；支持针对单个订单付款或一次性提交涵盖多个订单的付款凭证
- 为谁做：外部客户（买方）
- 解决什么问题：当前客户付款后需要通过邮件或聊天工具将水单发给业务员，业务员再手动录入系统；过程中容易遗漏、延迟，且水单格式五花八门难以归档管理

## 业务价值

- 对用户：客户在查看欠款的同时可直接提交付款信息，操作路径短、体验流畅
- 对业务：付款凭证结构化采集，减少业务员手动录入工作；水单自动归档，方便后续审计追溯
- 不做会怎样：客户只能看到欠款但无法在线响应，仍需依赖线下沟通完成付款通知，对账功能的价值大打折扣

## 验收标准

- [ ] 对账单页面每个未结清订单行提供"提交付款凭证"按钮
- [ ] 点击后弹出付款信息表单，包含：付款金额（必填）、付款日期（必填）、付款银行（选填）、银行流水号（选填）、备注（选填）
- [ ] 表单中提供水单图片/文件上传入口，支持 JPG、PNG、PDF 格式，单个文件不超过 10MB
- [ ] 付款金额不能超过该订单的欠款余额，超过时提示"付款金额不能超过欠款余额 XXX"
- [ ] 客户可以选择"合并付款"，即一笔付款对应多个订单，需勾选目标订单并输入总金额
- [ ] 提交成功后页面展示"付款凭证已提交，等待确认"的提示
- [ ] 提交成功后该订单行状态变为"凭证待审核"，直到内部确认后才更新收款金额
- [ ] 客户可以查看自己已提交的付款凭证历史列表（提交时间、金额、状态：待审核/已确认/已拒绝）
- [ ] 对于被拒绝的凭证，展示拒绝理由，客户可重新提交

## 影响范围

- 涉及模块：收款管理、销售订单、文件上传
- 涉及角色：外部客户

## 依赖关系

- 前置：#03 客户对账单外部查看页面（需要外部页面作为提交入口）
- 被依赖：#05 付款凭证审核与收款确认

## 约束条件

- 文件上传需做安全校验（文件类型白名单、文件大小限制、病毒扫描如可行）
- 同一订单在有"待审核"凭证时，不允许重复提交新凭证（避免重复付款混淆）
