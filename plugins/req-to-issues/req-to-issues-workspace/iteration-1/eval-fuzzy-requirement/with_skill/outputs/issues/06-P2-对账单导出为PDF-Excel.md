## 需求描述

- 做什么：业务员可以将某个客户的对账单导出为 PDF 或 Excel 文件，用于通过邮件等渠道发送给客户，或作为内部存档
- 为谁做：销售人员、CEO
- 解决什么问题：部分客户习惯接收文件附件形式的对账单，或者在正式商务沟通中需要带公司抬头的书面对账单；分享链接不适用于所有场景

## 业务价值

- 对用户：业务员可根据客户偏好选择分享链接或导出文件，沟通方式更灵活
- 对业务：导出的标准化对账单可作为商务往来的书面凭证，提升专业形象
- 不做会怎样：只有链接分享一种方式，无法满足需要正式文件的场景（如客户内部审批流程要求纸质/PDF 对账单）

## 验收标准

- [ ] 在客户应收账款页面，每个客户行提供"导出对账单"按钮，可选择 PDF 或 Excel 格式
- [ ] 导出的 PDF 包含：公司 Logo 和名称、客户名称、对账日期、订单明细表（订单号、日期、金额、已收、欠款）、汇总行、公司收款银行信息
- [ ] 导出的 Excel 包含与 PDF 相同的数据内容，格式规整，可直接打印
- [ ] 导出文件名格式：`对账单_客户名称_日期.pdf` 或 `.xlsx`
- [ ] 导出内容与在线对账页面数据一致
- [ ] 金额按币种分组展示，与在线页面逻辑一致

## 影响范围

- 涉及模块：收款管理、客户主数据、公司银行账户
- 涉及角色：CEO、销售

## 依赖关系

- 前置：#02 客户对账单生成与分享（复用对账单数据查询逻辑）
- 被依赖：无

## 约束条件

- PDF 模板风格与系统中已有的单证模板（Packing List、Commercial Invoice 等）保持一致
- Excel 导出复用现有模板机制（backend/templates/ 目录下的 xlsx 模板）
