## 需求描述

- 做什么：客户通过业务员分享的链接，无需登录即可查看自己的对账单，包括所有订单的应收金额、已收金额、欠款余额等明细信息，以及公司的收款银行账户信息
- 为谁做：外部客户（买方）
- 解决什么问题：客户目前需要反复联系业务员才能了解自己的欠款情况，信息获取不及时、不透明，影响客户付款积极性

## 业务价值

- 对用户：客户随时通过链接自助查看最新对账信息，无需等待业务员回复，体验类似在线账单
- 对业务：透明化欠款信息有助于缩短回款周期；减少业务员的重复沟通工作量
- 不做会怎样：对账单生成后客户无法查看，整个对账流程无法闭环

## 验收标准

- [ ] 客户点击分享链接后直接进入对账页面，无需登录或注册
- [ ] 页面顶部展示公司 Logo 和名称，以及该客户名称
- [ ] 展示汇总信息：订单总金额、已收总金额、欠款总金额（按币种分别展示）
- [ ] 展示订单明细列表：订单号、订单日期、订单金额、已收金额、欠款余额、收款状态（已结清/部分收款/未收款）
- [ ] 默认展示未结清订单，提供开关可查看所有订单（含已结清）
- [ ] 页面底部展示公司收款银行账户信息（账户名、银行名、账号、SWIFT 代码等）
- [ ] 链接过期后访问页面提示"对账单链接已过期，请联系业务员获取新链接"
- [ ] 链接被撤销后访问页面提示相同的过期信息
- [ ] 页面为只读展示，不包含任何修改数据的入口（付款凭证提交入口在后续 Issue 中添加）
- [ ] 页面适配移动端浏览器（客户可能在手机上查看）

## 影响范围

- 涉及模块：收款管理、销售订单、公司银行账户
- 涉及角色：外部客户

## 依赖关系

- 前置：#02 客户对账单生成与分享（需要分享链接和令牌验证机制）
- 被依赖：#04 客户在线提交付款凭证

## 约束条件

- 外部页面不经过系统登录认证，通过令牌验证身份，需确保令牌不可被暴力猜测
- 页面需考虑多语言展示（至少支持中英文），因为客户可能是海外买家
- 页面加载速度需在 3 秒以内（含数据查询）
