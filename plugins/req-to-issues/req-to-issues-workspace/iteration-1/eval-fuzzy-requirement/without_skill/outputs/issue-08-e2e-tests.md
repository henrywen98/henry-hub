# Issue #8: 客户对账与在线付款 - E2E 测试

## 类型
test

## 标签
`customer-portal`, `testing`, `e2e`, `medium-priority`

## 依赖
- Issue #4（客户门户前端页面）
- Issue #5（在线付款前端）

## 描述

### 背景
客户对账和在线付款涉及资金安全，必须有完善的端到端测试保障。需要覆盖客户门户的核心用户流程。

### 目标
使用 Playwright 编写端到端测试，覆盖客户门户的关键流程。

### 需求详情

#### 1. 客户登录流程测试
- `e2e/portal/login.spec.ts`
- 测试场景：
  - 正确凭证登录成功 → 跳转到门户首页
  - 错误密码登录失败 → 显示错误提示
  - 未登录访问门户页面 → 重定向到登录页
  - 客户 Token 访问内部管理页面 → 被拒绝
  - 员工 Token 访问门户页面 → 被拒绝

#### 2. 对账单查看流程测试
- `e2e/portal/statement.spec.ts`
- 测试场景：
  - 登录后查看对账总览，汇总数据显示正确
  - 对账单列表显示所有关联订单
  - 日期范围筛选正常工作
  - 仅有余额订单筛选正常工作
  - 对账单底部汇总与明细之和一致
  - 不同客户登录看到各自的数据（数据隔离验证）

#### 3. 在线付款流程测试
- `e2e/portal/payment.spec.ts`
- 测试场景：
  - 对有余额的订单发起付款（使用 Stripe 测试卡号）
  - 付款金额超过余额 → 验证阻止
  - 付款成功后余额更新
  - 付款失败场景处理
  - 付款记录出现在历史列表中

#### 4. 测试数据准备
- 创建测试种子数据脚本：
  - 测试客户及其门户账户
  - 该客户的多笔销售订单（不同状态、不同余额）
  - 历史付款记录
- Stripe 测试模式配置

### 技术要点
- 使用 Playwright 的 `test.describe` 组织测试套件
- 支付测试使用 Stripe 测试环境 + 测试卡号 `4242 4242 4242 4242`
- Stripe Elements iframe 交互需使用 `page.frameLocator()` 定位
- 测试前后清理测试数据，避免测试间相互影响

### 验收标准
- [ ] 登录流程 E2E 测试全部通过
- [ ] 对账单查看 E2E 测试全部通过
- [ ] 在线付款 E2E 测试全部通过（Stripe 测试模式）
- [ ] 测试可在 CI 环境中稳定运行
- [ ] 测试覆盖数据隔离场景（客户 A 看不到客户 B 的数据）
