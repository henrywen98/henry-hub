# 交叉核验清单

文档生成后、交付用户前，执行以下自动验证。

---

## 模块覆盖率检查

1. 从 Phase 1 中记录的模块名列表（PRD 中检测到的 N 个模块）
2. 用 Grep 在生成的概设文件中搜索每个模块名
3. 对比确认：每个 PRD 模块在概设中都有对应的 `## 3.x` 章节
4. 如有遗漏，报告缺失的模块名

## 章节占比检查

1. 用 Grep 定位概设中各一级章节标题的行号
2. 计算第3章（功能模块设计）的行数占总行数的比例
3. 如果低于 60%，报告警告

## 一致性检查

1. 检查汇总表中的模块列表与第3章各模块标题是否一致
2. 检查编号规则汇总中的模块是否都在功能设计表格中出现
3. 检查是否存在 `<!-- TODO -->` 标记，逐一报告

## 名称对齐检查

1. **全景图 vs PRD**：逐一对比全景图中每个模块名称与 PRD 原文标题，不允许缩写（如 PRD 写"采购申请"则不能简写为"采购"）
2. **全景图 vs 第3章**：全景图中每个模块在第3章中都有对应章节标题，名称一致
3. **全景图模块组完整性**：如果存在多个模块组，每个组的列必须完整列出该组拥有的所有模块，不允许用"复用XX其余模块"等模糊表述

## 共享模块声明核验（仅多模块组时执行）

1. 检查"共享模块列表"中声明的每个模块，是否在 PRD 的每个模块组中都确实存在
2. 检查差异表是否覆盖了所有实际存在的差异（模块有无差异 + 业务逻辑差异）
3. 如发现共享声明与 PRD 不符，报告并修正

## Copy-paste 错误检测

1. 扫描审批流程名称：检查流程名称中的模块关键词是否匹配所在章节（如"退出管理"模块的流程名称不应含"立项申请"）
2. 扫描事项名称默认值：检查默认生成的事项名称是否与当前模块匹配

## 第2/5章去重检查

1. 对比第2章和第5章的内容，检查是否有大段重复的表格或描述
2. 第5章中与第2章重复的部分应使用"详见第 2.x 节"引用

## 核验报告模板

向用户展示核验结果：

```
✅ 模块覆盖率：N/N（100%）
✅ 第3章占比：XX%（达标/未达标）
✅ 汇总表一致性：通过
✅ 全景图名称对齐：通过
✅ 全景图模块组完整性：组A M/M，组B K/K（仅多模块组时显示）
✅ 共享模块声明：N 个共享模块均核验通过（仅多模块组时显示）
✅ Copy-paste 检测：未发现异常
✅ 第2/5章去重：无重复段落
⚠️ TODO 标记：0处
```

如果有问题，主动修复后再交付。
