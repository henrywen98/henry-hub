# 交叉核验清单

文档生成后、交付用户前，执行以下自动验证。

---

## 模块覆盖率检查

1. 从需求确认单中检测到的模块名列表
2. 用 Grep 在生成的概设文件中搜索每个模块名
3. 对比确认：每个需求模块在概设中都有对应的 `### 3.2.x` 章节
4. 如有遗漏，报告缺失的模块名

## 章节占比检查

1. 用 Grep 定位概设中各一级章节标题的行号
2. 计算第3章（功能模块设计）的行数占总行数的比例
3. 如果低于 60%，报告警告

## 结构完整性检查

1. 7 章全部存在，无缺章
2. 每章所有子节全部存在
3. 每个子系统都有对应的功能表格
4. 功能表格每行四列都有内容

## 一致性检查

1. 功能结构图中的子系统 = 3.2.x 中的子系统一一对应
2. 如有硬件表，服务器总数 = 部署设计中描述的总数
3. 技术选型与技术路线中的技术栈一致
4. 检查是否存在 `<!-- TODO -->` 标记

## 名称对齐检查

1. 概设中每个模块名称与需求确认单原文一致，不允许缩写
2. 功能结构图中每个模块在第3章中都有对应章节标题

## Copy-paste 错误检测

1. 检查是否有从过往概设复用时未替换的项目名称/公司名称
2. 检查第3章功能表格中是否有与当前项目不匹配的内容

## 第2/5章去重检查

1. 对比第2章和第5章，检查是否有大段重复的表格或描述
2. 第5章中与第2章重复的部分应使用"详见第 2.x 节"引用

## 核验报告模板

```
✅ 模块覆盖率：N/N（100%）
✅ 第3章占比：XX%（达标/未达标）
✅ 结构完整性：7章齐全，子节完整
✅ 一致性：通过
✅ 名称对齐：通过
✅ Copy-paste 检测：未发现残留的旧项目信息
✅ 第2/5章去重：无重复段落
⚠️ TODO 标记：0处
```

如有问题，主动修复后再交付。
