# concept-design — 概要设计说明书组装器

从需求确认单和过往概设模板组装面向甲方交付的概要设计说明书。

## 核心理念

概设是合同要求的交付物，不是技术创作。功能范围在需求确认单中已经确认，技术/安全/部署章节在项目间变化不大。所以概设的工作本质是**格式转换 + 模板复用**。

```
需求确认单 ──→ 第3章功能模块（格式转换）
过往概设模板 ──→ 第1/2/4/5/6/7章（复用微调）
```

## 输入素材

| 素材 | 必需性 | 用途 |
|------|--------|------|
| 需求确认单 / PRD | **必须** | 第3章功能模块的来源 |
| 过往概设 / 概设模板 | **强烈建议** | 其他所有章节的骨架和内容来源 |
| 技术架构文档 | 可选 | 有架构变化时用来更新第2/5章 |
| 原型文件 | 可选 | 辅助第5.5章界面设计描述 |

所有素材放在当前工作目录下即可。

## 输出

| 章节 | 来源 | skill 做的事 |
|------|------|-------------|
| 第3章 功能模块设计（60%+） | 需求确认单 | 格式转换：功能列表 → 概设四列表格 |
| 第2/5章 技术方案 | 过往概设 | 微调复用：换项目名，调整细节 |
| 第6章 接口 | 需求确认单 + 过往概设 | 提取+套框架 |
| 第7章 安全 | 过往概设 | 直接复用 |
| 第1/4章 引言/非功能 | 过往概设 | 直接复用，换项目名 |

## 使用方式

```
生成概设
写概要设计
概要设计说明书
```
