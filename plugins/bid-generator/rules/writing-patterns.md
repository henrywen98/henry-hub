# 各类章节写作模式

## 使用说明

本文件定义了投标文件各类章节的写作模式。
被 /bid.fill skill 引用，指导 LLM 按正确的结构和语气生成内容。

共 7 种模式，从两份历史标书（沈阳盛京、国家制造业）中归纳并经过实际内容验证。
每种模式包含：结构定义、写作要点、从知识库提取的示例开头段落。

---

## 模式1：架构设计类

适用章节：总体架构设计、技术架构设计、数据架构设计、业务逻辑架构设计、部署架构设计

### 结构

```
建设原则 → 技术选型(含选型理由) → 架构概述 → 架构图描述(分层) → 各层能力说明
```

### 要点

- 技术选型在架构概述之前（先明确技术栈再展开架构）
- "方案优势"不单独成段，融入各技术组件说明中
- 需包含"业务逻辑架构"（业务流图、数据流图）
- 须包含至少1张架构图（ASCII art 文本框图），展示分层架构、核心组件和数据流
- 架构图放在"架构概述"之后，图下方紧跟各层能力的文字说明
- 语气：专业技术文档，强调先进性和成熟度

### 示例开头

> 系统技术架构以"微服务为核心、前后端分离为基础、低代码赋能灵活扩展"为核心定位，严格遵循"高内聚低耦合、可配置可扩展、安全可靠、性能优越"四大设计原则。整体架构通过微服务拆分实现业务解耦，结合前后端分离模式提升开发效率与用户体验，依托低代码平台及配套工具支撑业务快速定制与迭代。

（来源：`knowledge/02-总体方案/技术架构设计.md`）

## 模式2：功能模块类

适用章节：股权投资管理、基金管理、风控合规、监管报送、数据驾驶舱与报表、统一门户与工作台、文档与档案管理、系统集成、AI智能能力、相关方管理等

### 结构

```
功能概述(1-2段) → 直接进入子功能分层详述
每个子功能：功能说明 → 核心操作流程 → 关键特性
```

### 要点

- 不需要独立的"业务流程总图"和"功能清单表"
- "亮点创新"不在功能章节内，作为独立评分应答章节
- 招标需求中的关键术语必须在正文中原文复现
- 语气：以业务价值为导向，兼顾技术实现

### 示例开头

> 股权投资管理是投资管理信息系统中最核心的功能模块，覆盖股权投资从投前、投中到投后的全生命周期管理。通过该模块实现投资计划的下达与执行、项目库的全流程管理、控股股权投资（直投）的差异化流程配置，以及投后阶段的一企一档式管理。

（来源：`knowledge/03-功能模块/股权投资管理.md`）

## 模式3：实施方案类

适用章节：项目组织与沟通、实施方法论、进度与里程碑、测试方案、验收方案、规范性保障、风险管控

### 结构

```
项目组织机构(决策/实施/验收层) → 沟通机制 →
方法论(CMMI/PMBOK) → 阶段划分 → 里程碑(表格) → 交付物清单 →
测试方案(测试管理/自动化测试/性能测试) → 验收方案(初验/终验流程) →
质量保证 → 实施规范性保障(进度控制/需求变更/配置管理/代码管理) →
风险管控(管理/技术/人员三类)
```

### 要点

- 方法论引用成熟框架（CMMI5、ISO9001、OSPM等）
- 阶段划分须与招标要求的工期对齐
- 里程碑表格包含：阶段、时间、交付物、验收标准
- 测试方案占比较大，需详细展开
- 语气：强调可控性和经验丰富

### 示例开头

> 作为项目管理通过CMMI5认证的软件公司，基于CMMI5管理体系下持续改进，不断调整最终形成项目管理体系，它由"启动"、"策划"、"执行"、"管理和监控"、"收尾"5个管理过程组组成，项目管理过程活动按照这5部分有序开展。

（来源：`knowledge/05-实施方案/实施方法论.md`）

## 模式4：维保方案类

适用章节：服务范围与内容、定期巡检计划、故障处理与响应、质量保证措施、应急预案、维保团队

### 结构

```
服务范围(总述) → 定期巡检计划(目标/周期/内容/流程) →
故障处理(分级响应,含响应时间承诺) → 系统优化升级 →
质量保证措施(环境/安全/流程/工具) →
应急预案(灾备切换/业务连续性) → 维保团队(资格/考核)
```

### 要点

- SLA 指标分散在"巡检"和"故障处理"中，不单独成节
- SLA 指标须大于等于招标要求
- 响应流程分级：一般/紧急/重大
- 语气：强调承诺和保障力度

### 示例开头

> 系统投入运行后进入质保阶段，质保阶段的主要任务是保证系统正常运行，同时对系统运行状况进行监测和诊断，及时排除各种故障，在必要时对系统进行升级。维护方式包括电话解答、远程维护、技术人员现场解决问题三种形式，保证提供每周7x24小时不间断的售后服务。

（来源：`knowledge/06-维保方案/服务范围与内容.md`）

## 模式5：培训方案类

适用章节：培训方案、售后服务、知识转移

### 结构

```
培训目标 → 培训对象分层(管理层/操作层/运维层) →
培训大纲(初级/高级) → 培训计划(时间/地点/方式) →
知识转移方案 → 培训考核
```

### 要点

- 培训对象分层须与招标文件中的角色/岗位对应
- 培训时长须大于等于招标要求（如"操作培训10人天"）
- 知识转移方案需说明文档交付和技术交底流程
- 语气：强调体系化和可落地

### 示例开头

> 培训工作需要贯彻始终，并且对全体用户进行全方位的培训。由于培训任务涉及面广、人员众多，培训工作应照顾现有的工作模式，同时针对不同的培训对象进行分层培训。培训方式灵活多样，包括一对一培训、集中培训等，不限人数和次数。

（来源：`knowledge/07-培训与售后/培训方案.md`）

## 模式6：公司与案例类

适用章节：公司资质、项目案例、核心团队

### 结构

```
公司简介 → 核心资质(CMMI等) →
同类案例(表格,满足招标要求的数量/时间/行业) →
重点案例详述(2-3个: 背景→内容→成果) →
核心团队(项目经理/技术经理/开发人员)
```

### 要点

- 案例需满足招标要求（行业、时间范围、数量）
- 案例和团队标记为需更新（生成时提醒用户确认）
- 语气：强调实力和经验匹配度

### 示例开头

> 璞华科技通过 CMMI Level 5（持续优化级）认证，代表其软件开发与项目管理能力达到全球最高成熟度级别。在开发过程中严格遵循 CMMI5 规范，通过流程控制和监管减少开发风险，为客户提供优良的软件产品。公司拥有软件著作权320余项、核心专利56件。

（来源：`knowledge/08-公司与团队/公司资质.md`）

## 模式7：偏离表类

适用章节：技术偏离表、商务偏离表

### 结构

```
逐条列出招标要求 → 对应"满足/优于/偏离" → 偏离项说明替代方案
默认策略：除非知识库明确不支持，否则填"完全满足"
```

### 要点

- 技术偏离直接影响评分（每条负偏离扣分）
- 商务实质性条款（服务期、付款方式）偏离将导致废标
- 正偏差可展示技术优势和附加价值，不扣分
- 负偏差必须明确标注，说明替代方案和解决时间表
- 逐条对照招标文件技术需求书，确保不遗漏
- 语气：严谨、明确、无歧义

### 示例开头

> 偏离表是投标文件的核心组成部分，用于对比投标人的技术方案和商务条款与招标文件要求之间的差异。投标人应尽量做到"完全响应，无偏离"。技术条款偏离直接影响评分，商务条款中服务期和付款方式为实质性条款，偏离将导致废标。

（来源：`knowledge/10-程序与资格文件/偏离表模板.md`）
