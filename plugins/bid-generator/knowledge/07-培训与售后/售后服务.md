# 售后服务

## 一、售后服务体系总述

致力于成为客户支持与服务方面的首选合作伙伴，为此建立了强有力的、完善的支持服务体系，并获得专业的信息技术服务标准ITSS服务认证。服务体系的核心优势包括：

- **完善的专业服务人员配备**：响应及时的一线工程师，解决疑难问题的专业升级工程师，专业化的服务网络管理/运作系统。
- **强大的技术支持体系**：对工程师进行系统、多层次、周期频繁的培训，工程师必须认证上岗；研发中心的资深工程师和产品工程师为实施工程师提供强有力的技术后盾。
- **多样化的远程诊断系统**：热线电话支持、在线支持，以最快的速度排忧解难。
- **全面质量监控**：建立完善的维修运作标准，运用客户调查、综合业绩测评等办法进行严格的质量监控。

## 二、免维期

### 2.1 沈阳盛京项目

自项目验收合格之日起，提供36个月的免费维护服务。免费维保期限满后的年维保服务费不超过项目建设费用的10%。

### 2.2 国家制造业项目

自项目验收合格之日起，提供48个月的免费维护服务，包含整体系统（软硬件）免费维保服务。另承诺在1年整体系统免费维保服务的基础上，提供3年的免费维保，以及5年免费技术升级服务。免费维保期限满后的年维保服务费为项目建设费用的8%（228,000元/年，承诺5年整体续保报价不变）。

### 2.3 免维期通用服务内容

- 在免费维护期内提供免费平滑维护，不再另外收取差价或费用。
- 服务内容包括：使用指导，对系统进行必须的修订，对系统隐含的错误进行纠正。
- 维护方式：电话解答、远程维护、技术人员现场解决问题。对故障在30分钟内响应，2小时以内提出问题解决方案并采取相应措施保障用户正常使用，4小时内到达现场。
- 具备专业的服务团队，保证提供每周7x24小时不间断的售后服务。
- 对于系统的BUG承诺终身免费维护，定期汇报系统运行情况。

## 三、软件升级方案

软件升级主要包括系统的安装以及升级服务。需要根据实际情况以及用户需求，在不影响用户正常使用的情况下，定期或不定期进行升级维护服务。产品在质保期提供小版本免费升级服务。

### 3.1 安装服务

正确的产品安装和配置是正常使用和发挥其性能的基本前提。在安装设备之前，需先对用户进行现场培训；安装过程中，需要相关技术人员参与安装、配置、检测和排除故障。

### 3.2 备份服务

在实施升级服务之前，必须对系统及数据进行备份。由于升级过程中可能因升级版本的不稳定因素造成系统运行中断等严重故障，在此情况下可直接恢复为原稳定版本以保证系统正常运行。

### 3.3 升级服务流程

1. **升级服务请求**：用户根据实际需求提出升级要求，填写升级请求单，交由项目经理审核。
2. **升级服务审核**：项目经理审核通过后，组织相关技术实施人员完成升级部分的功能开发、设计、测试，最终形成升级版本，下发至升级维护组；如果升级服务请求没有通过审核，则直接反馈给用户。
3. **升级服务实施**：确认升级版本之后，将现有系统及数据进行备份，再进行升级操作。如果升级成功则着重对更新版本进行跟踪以确认稳定性；如出现升级失败的情况，需尽快恢复升级前的备份版本并保证正常运行，由相关技术实施人员重新对升级部分进行测试评估。

## 四、维修维护方案

### 4.1 保修期内的服务

- **驻场维保服务**：对保修期内的客户业务系统提供驻场维保服务，包括现场勘查、了解客户业务流程与体制、提供项目方案实施咨询服务。
- **安装服务**：包括主系统安装（操作系统、数据库系统、应用软件、服务器端软件联调）和客户端配置。
- **技术服务**：平台升级服务、性能优化服务、日常系统及服务器运维服务、系统和中间件安全漏洞修复升级。
- **文档服务**：项目进行过程中提供文档整理服务（项目资料、项目报告、程序整理），项目结束时提供管理员手册和维护手册。
- **培训服务**：使用人员培训（软件流程、软件操作）和管理员培训（软件流程、操作及参数设置、日常维护）。
- **保驾服务**：提供保驾服务以保证系统7x24小时的稳定运行和系统切换的顺利进行。

### 4.2 保内服务补充说明

- 保内服务包括用户在使用中发现系统软件问题而需要进行小版本的升级、维护服务。
- 保内服务包括用户在正常使用状态下系统出现故障、紧急事故而引起的紧急维护服务。
- 保内服务不包括因用户使用与安全防范问题造成病毒泛滥所引起的维护与重新安装服务。
- 保内服务不包括因用户擅自对系统修改、删除及进行二次开发、安装第三方软件造成系统故障所引起的维护与重新安装服务。

### 4.3 保修期外的服务

- **BUG免费终身维护**：对于系统的BUG提供免费终身维护（现有技术无法解决原因造成的除外）。
- **热线服务**：通过电话向用户提供7x24小时技术问题解答。
- **年保服务**：通过年度维护保障服务合同（IT外包）方式和客户确定服务的具体项目和级别，保证客户的业务系统稳定运行。

## 五、本地化服务承诺

为保障项目在质保期及后续服务期内能够持续、稳定、高效地运行，提供以下本地化服务：

- **故障分级响应现场承诺**：根据故障等级在约定时间内到达现场。
- **现场用户培训与交流**：在客户现场进行培训和技术交流。
- **本地团队版本升级与补丁部署支持**：由本地团队完成版本升级与补丁部署。
- **个性化服务定制**：根据客户具体需求提供定制化服务。
- **本地资源保障**：维保阶段提供至少1名技术服务人员，保障本地技术团队的人员稳定性。
- **定期技术交流**：不定期进行技术交流，形式可以是座谈、讲座和培训。
- **电话回访**：主动不定期通过电话对项目使用单位及管理单位进行回访。
- **满意度调查**：定时向客户征询满意度调查，以持续提升服务质量。

### 免维期详细承诺

根据项目规模提供差异化免维期承诺：
- 标准免维期：项目验收后12个月
- 增强免维期：1年基础免维 + 3年额外免费维护服务，共计48个月
- 免费技术升级服务：5年内系统功能升级免费提供
- BUG终身免费维护：系统自身缺陷终身免费修复

### 运维管理体系四个专项小组

- **日常维护组**：负责日常巡检、用户问题响应、数据核查、系统监控
- **升级维护组**：负责需求变更评估、功能开发、版本管理、升级部署
- **技术维护组**：负责数据库维护、中间件维护、服务器维护、性能优化
- **应急支援组**：负责重大故障应急处理、灾难恢复、数据抢救

### 升级服务流程

用户提出升级请求 → 项目经理审核评估（工作量/影响范围/排期）→ 技术人员完成开发和内部测试 → 确认升级版本和发布说明 → 生产环境备份 → 升级操作执行 → 跟踪确认运行稳定性（至少观察3个工作日）。重大升级需与用户协商升级窗口期，提供回退方案。
