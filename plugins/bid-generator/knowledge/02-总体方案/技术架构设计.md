# 技术架构设计

## 概述

系统技术架构以"微服务为核心、前后端分离为基础、低代码赋能灵活扩展"为核心定位，严格遵循"高内聚低耦合、可配置可扩展、安全可靠、性能优越"四大设计原则。整体架构通过微服务拆分实现业务解耦，结合前后端分离模式提升开发效率与用户体验，依托低代码平台及配套工具支撑业务快速定制与迭代。

## 详细内容

### 一、技术架构总体设计

系统整体采用微服务架构、前后端分离开发模式，业务功能基于微服务开发平台、低代码平台以及其他配套工具进行开发，实现业务的灵活定制。系统遵循规范化、自主可控原则，系统框架技术确保先进成熟、自主可控；系统架构中各层采用成熟的、符合技术标准的国产化服务器、数据库产品。

系统支持在国产化信创运行环境部署，符合保密和安全管理规定，对部分信息加密存储；具备高性能高可用的部署方案和灾难恢复解决方案；支持弹性、动态的部署架构，支持平顺的软硬件升级和扩展，保证核心应用系统的独立性和可扩展性。通过服务熔断、限流、分布式事务、多活部署等机制，保障系统高可用与数据一致性。

在技术指标上满足等保三级国家信息安全体系管理要求、软件测试和密评要求，具备通过等保三级国家信息安全体系评测、软件测试和密评的能力。

### 二、后端技术栈

系统后端采用Java作为基础开发语言，基于JDK 1.8版本开发，设计及开发符合J2EE体系结构及规范。Java是目前主流编程语言，J2EE体系及Spring生态体系具有完善的二次开发平台。

后端应用服务采用基于Spring Boot、Spring Cloud、Spring Cloud Alibaba的微服务框架，实现页面和具体业务的快速开发。核心技术选型如下：

| 领域 | 技术 | 技术优势 |
|------|------|----------|
| JDK选型 | JDK 1.8 64位 | 迄今为止最稳定、用户最多的版本 |
| 应用服务层 | Spring Boot、Spring Cloud | 提供大型项目中常见的非功能性特性，如嵌入式服务器、安全、指标、健康检测、外部配置等 |
| 注册中心/配置中心 | Nacos | 提供动态配置服务，友好的运维管理界面 |
| 分布式事务 | Seata | 高性能、简单易用、支持高可用 |
| 消息中间件 | RocketMQ | 基于消息机制，提供存储及传递消息的功能，高效解耦业务系统 |
| 定时任务 | XXL-JOB | 开源轻量级分布式任务调度平台，开箱即用 |
| 规则引擎 | Drools 7 | 提供开放的封装业务规则的计算组件 |
| 工作流引擎 | 基于BPMN 2.0 | 对工作流程可动态配置，提供基于浏览器的流程设计器 |

采用Spring Cloud Gateway作为网关基础组件，采用Nacos作为注册中心和配置中心，使用Seata作为分布式事务组件。采用前后端分离的开发方式，前后端通信使用HTTP/HTTPS+JSON的方式，针对老系统可开发单独的接口，定义针对性的接口规范，支持XML数据交换标准。

### 三、前端技术栈

前端工程包括移动端的前端工程全部采用H5 + Vue 3 + Vue-Router + Pinia + Webpack技术组成，配合Element、Vant等UI组件库完成前端项目的开发。前端环境主要依赖Node.js运行环境，依赖包使用npm安装，通过Nexus私服仓库中建立的npm私服加快前端依赖下载速度。

前端的开发环境在开发机器上安装Node.js运行环境，并将npm仓库切换为Nexus中的npm私服。其中Node.js环境使用NVM工具来安装，便于统一管理和切换Node.js版本。前端的测试环境、生产环境不需要在服务器上安装Node.js环境，只需要在Jenkins中安装对应版本即可。

### 四、移动端技术路线

移动端平台包括两部分：一是基于UniApp的基座应用，考虑到其高性能、接近原生App应用的用户体验流畅度，同时可打包生成各系统适配应用；二是基于H5的业务应用，主要用于实现各功能业务操作和展示。

基于UniApp框架进行移动端开发，具有跨平台兼容性，编写一次代码并多次部署到Android、iOS、HarmonyOS操作系统。原则上除系统后台配置功能外，移动端具备所有功能；支持手机、平板等移动端应用；支持移动门户展示、移动端工作消息提醒、在移动端发起流程和审批流程等。

### 五、国产信创适配选型

#### 1. 操作系统

建议采用麒麟V10服务器操作系统。银河麒麟云原生操作系统V10面向云原生场景，采用"内核+操作系统+Kubernetes"联合设计思想，支持操作系统集群粒度统一纳管、一致性发布和原子升级回滚，对容器调度、容器网络、容器运行时等关键组件进行深入优化。

#### 2. WEB应用服务器

为满足中间件信创要求，系统应用服务器采用嵌入式TongWeb 8.0，同时支持Java EE企业级标准与React响应编程标准，通过集成Spring Boot、Spring Cloud框架使用。以内嵌TongWeb 8的jar包方式部署运行。

#### 3. 数据库

数据库推荐达梦数据库DM8，具有完全自主知识产权的国产高性能数据库产品，严格遵循国际SQL标准，在大数据量处理、高并发访问、高可用性和安全性等方面进行了深度优化。核心优势包括高性能、高可扩展性、高可靠性、高安全性及良好兼容性。

#### 4. 负载中间件

使用Tengine作为负载中间件。Tengine基于Nginx，具备性能优化、模块化设计、安全性增强、运维友好、稳定性提升和云环境适配等核心特性，适用于企业级网站、CDN服务、API网关和微服务架构等场景。

#### 5. 分布式缓存

使用TongRDS作为分布式缓存中间件，为各类应用提供高效、稳定、安全的内存数据处理能力，支持共享内存的弹性伸缩管理。实现热点数据的高速读取，同时作为用户认证存储认证信息，实现Session共享。访问频率较高的业务数据使用TongRDS分布式缓存。

#### 6. 消息中间件

使用RocketMQ作为消息中间件，实现消息队列、异步调用、流量消峰。完全支持国产化且实现技术自主可控，支持国产操作系统（麒麟、欧拉、统信UOS）和国产芯片架构（鲲鹏、飞腾、龙芯），可与国产数据库无缝集成。

#### 7. 分布式存储

采用OSS作为分布式存储服务，支持存储非结构化数据（照片、视频、日志文件、备份等），具备高性能、高可用性、可扩展性和安全性。

### 六、其他关键技术组件

**报表应用技术：** 基于开源积木报表的企业级Web报表工具，通过拖拽完成报表设计，支持SQL和API两种数据源，支持ECharts图表（28种），满足数据报表、打印设计、图表报表、大屏设计等需求。

**业务流程管理：** 系统表单、流程、数据等通过图形化配置构建，低代码或无代码开发，满足系统后续的扩展和外延。支持可视化的流程自定义功能，包含自由流、子流程、各种条件判断等复杂业务流，提供数字化的企业级业务流程管理能力。
