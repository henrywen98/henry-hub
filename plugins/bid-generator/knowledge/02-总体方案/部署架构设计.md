# 部署架构设计

## 概述

投资管理信息系统基于微服务架构实现，采用高可用集群部署方式，支持分布式部署和集群发布，所有节点高可用部署，保证无单点故障。系统的部署支持多种方式，如云原生、私有化、容器化等，具备灵活扩展的能力，可随着资源规模的增长快速扩展，扩展过程中不会对平台操作造成影响，支持滚动升级，保障业务的连续性。

## 详细内容

### 一、部署架构概述

系统架构及部署方案具备7x24小时运行能力，具有异常或故障处理能力，并具有启停、续接服务机制。支持多服务器集群部署和虚拟化平台部署，实现自动负载均衡。支持多点部署、分布式计算、分布式存储、按需分配、弹性扩展的物理架构，提供统一的系统接口服务。

系统具备良好的环境适配性，支持在不同技术架构下进行灵活部署。优先适配云空间及纯国产信创机房两种环境，形成差异化部署方案。支持运行在采用Intel、AMD等CPU的X86服务器上，也支持运行在采用飞腾、鲲鹏等CPU的信创服务器上。支持私有云环境部署，同时具备支持公有云环境部署的能力。

### 二、多环境部署方案

系统包括开发、测试、生产三套环境，为系统部署搭建全套DevOps自动化管理工具，覆盖需求管理、计划管理、代码管理、持续集成、缺陷管理、自动化测试、持续部署、持续交付流水线、可视化研发管理仪表盘等功能，通过管理工具实现应用在各环境的快速发布。

#### 1. 开发环境

开发环境采用容器化的部署方式，便于开发人员快速搭建和调试。前端开发环境在开发机器上安装Node.js运行环境，后端开发环境安装JDK即可。中间件有些需要本地运行，有些可以安装在服务器上供开发环境共同使用。数据库按照开发环境独立安装一套。

#### 2. 测试环境

测试环境同样采用容器化部署，模拟生产环境的配置进行功能测试和性能测试。前端服务器同时做构建服务器，安装Jenkins、Maven、GitLab进行DevOps自动化部署和自动化测试。

#### 3. 生产环境

生产环境在内网私有化环境中采用高可用集群部署方式，也支持容器化和云原生方式。生产环境的服务器配置按3-5年规划，满足用户的业务使用需求，服务器配置可以根据业务量进行扩展和缩减。

### 三、服务器部署拓扑

根据技术架构和部署架构，将服务器划分为Web接入服务器、应用服务器、中间件服务器、数据库服务器和文件服务器。为满足系统性能、高可用、高并发、扩展性的需求，设计如下部署方案：

#### 1. Web接入服务器

部署Tengine及前端静态包，作为统一的接入入口，处理HTTP/HTTPS请求转发和静态资源服务。Web接入服务器仅用于部署Tengine及前端静态包，通过Tengine软负载能方便快速地进行集群搭建，以快速响应业务的增长。

#### 2. 应用服务器

部署微服务应用实例，包括各业务模块的Spring Boot服务。应用服务器兼做中间件服务器，部署Nacos注册中心、RocketMQ消息队列、XXL-JOB定时任务等中间件服务。多台应用服务器做中间件集群，保证所有中间件的高可用。

#### 3. 数据库服务器

部署达梦DM8数据库实例，采用主备集群模式，数据库服务器为两台集群做主备服务，保证数据的高可用和安全性。提供数据的读写分离能力，提升查询性能。

#### 4. 文件服务器

部署OSS分布式存储服务，管理系统中的各类非结构化文件资源。文件服务器兼做数据库服务器的从机，数据库服务器兼做文件服务器的备份服务器，实现存储资源的互备。

### 四、部署方案说明

整体部署系统是多个Spring Boot子项目，搭配注册中心、缓存服务、日志服务、定时任务等关联服务，组合成的一整套基于Spring Cloud的微服务平台。

整个部署架构符合网络拓扑结构，不同的服务器分别在不同的网络区域中，确保网络安全隔离。关于分布式缓存代替集中式缓存，提高业务数据查询效率，减少宕机率，保证数据的完整性。

前端服务器同时做构建服务器，安装Jenkins、Maven、GitLab进行DevOps自动化部署、自动化测试，实现持续集成和持续交付。

### 五、高可用与容灾设计

#### 1. 应用层高可用

所有应用服务节点采用集群部署，通过Tengine负载均衡实现请求分发和故障转移。当某个服务节点出现异常时，负载均衡器自动将请求转发到其他健康节点，确保服务不中断。微服务架构支持服务的独立部署和滚动升级，单个服务的升级不影响其他服务的正常运行。

#### 2. 数据层高可用

数据库采用主备集群模式，主库负责数据写入，备库同步数据并提供读取服务。通过数据镜像和故障切换机制，在主库出现故障时自动切换到备库，保证数据服务的连续性。同时每天定时数据异地备份上传，保证数据不出现丢失。

#### 3. 中间件高可用

注册中心（Nacos）、消息队列（RocketMQ）、分布式缓存（TongRDS）等关键中间件均采用集群部署，支持多台服务并行运行，各服务可互为主备以负载分担方式运行。数据采用独立队列完成多服务器间同步，同步异常不影响主业务。服务重启时可从其他正常的服务获取在线数据，单台服务故障重启不影响整个系统正常提供业务。

### 六、系统运行环境

系统支持主流平台操作系统，支持国产操作系统、数据库、中间件。系统架构支持PC端、平板电脑、移动端上市场占有率较高的各类浏览器。支持移动终端运行，支持对移动应用的单个安装、卸载、停用和重启。

运行环境要求如下：

| 组件 | 要求 |
|------|------|
| 操作系统 | 国产化OS：华为欧拉、麒麟V10、统信V20等 |
| 开发语言及版本 | Java JDK 1.8 |
| 数据库 | 达梦数据库DM8 |
| 应用服务器 | TongWeb 8 |
| 中间件 | Tengine、TongRDS、Nacos、RocketMQ、Seata |

系统支持在不同技术架构下进行灵活部署，系统采用的软件开发技术都属于开源系统项目，软件系统可运行于通用的主流硬件平台上，不依赖于特定的、专用的硬件设备或者系统软件。系统配置（硬件系统、操作系统、数据库系统）的升级一般情况下不会引起系统的修改和再次开发。
