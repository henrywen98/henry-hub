# 偏离表模板

## 一、概述

偏离表是投标文件的核心组成部分，用于对比投标人的技术方案和商务条款与招标文件要求之间的差异。偏离表分为商务条款偏离表和技术规格偏离表两类。

**重要提示**：
- 技术条款偏离直接影响评分（如国家制造业项目：无负偏离得5分，每有一条负偏离扣1分）
- 商务条款中服务期和付款方式为实质性条款，偏离将导致废标
- 投标人应尽量做到"完全响应，无偏离"

## 二、商务条款偏离表

### 模板格式

```
商务条款偏离表

项目名称：{项目全称}
招标编号：{招标编号}

| 序号 | 招标文件的商务条款 | 投标文件的商务条款 | 说明 |
|------|------------------|------------------|------|
| 1    | 见招标文件的所有商务条款 | 见投标文件的商务条款 | 完全响应，无偏离。 |
|      |                  |                  |      |

投标人授权代表姓名（签字或盖章）：
投标人名称：{公司全称}（公章）
日期：{年} 年 {月} 月 {日} 日
```

### 填写说明

1. **商务条款范围**：包括但不限于"合同条款"部分中的所有条款
2. **实质性条款**：服务期和付款方式为实质性条款，偏离将被作为无效投标处理
3. **无偏离情况**：如投标人完全响应所有商务条款，只需填写一行"完全响应，无偏离"
4. **有偏离情况**：须详细描述偏离内容，仅列出偏离项
5. **不响应处理**：如投标人不响应该部分内容，视同无偏离

### 有偏离时的详细格式

```
| 序号 | 招标文件的商务条款 | 投标文件的商务条款 | 说明 |
|------|------------------|------------------|------|
| 1    | {招标文件原文} | {投标人的响应内容} | {正偏差/负偏差/完全满足} |
| 2    | {招标文件原文} | {投标人的响应内容} | {偏差说明} |
```

## 三、技术规格偏离表

### 模板格式

```
技术规格偏离表

项目名称：{项目全称}
招标编号：{招标编号}

| 序号 | 招标文件的技术条款 | 投标文件的技术响应 | 应答结果 | 说明 |
|------|------------------|------------------|---------|------|
| 1    | 见招标文件技术条款 | 见投标文件技术响应 | 满足 | 完全响应，无偏离。 |
|      |                  |                  |         |      |

投标人授权代表姓名（签字或盖章）：
投标人名称：{公司全称}（公章）
日期：{年} 年 {月} 月 {日} 日
```

### 应答结果类型

| 结果 | 说明 |
|------|------|
| **满足** | 完全满足招标文件的技术条款要求 |
| **正偏差** | 在完全满足技术条件的基础上，更加充足的应答条件 |
| **负偏差** | 部分满足技术条件需求，其中未能满足的应明确列明 |
| **不满足** | 无法满足该项技术条款要求 |

### 有偏离时的详细格式

```
| 序号 | 招标文件的技术条款 | 投标文件的技术响应 | 应答结果 | 说明 |
|------|------------------|------------------|---------|------|
| 1    | {功能模块A要求：具体内容} | {我方响应：具体方案} | 满足 | 完全满足 |
| 2    | {功能模块B要求：具体内容} | {我方响应：提供增强功能} | 正偏差 | 在满足基础上额外提供{增强功能说明} |
| 3    | {功能模块C要求：具体内容} | {我方响应：部分满足} | 负偏差 | {未满足部分}需在{时间}内通过{方式}实现 |
```

### 投标人声明

技术规格偏离表末尾应附上声明：

```
注：
1. "应答结果"填写：满足、正偏差、负偏差、不满足。其中，填写正偏差或负偏差的，请在"说明"中予以说明。
   - 正偏差：在完全满足技术条件的基础上，更加充足的应答条件；
   - 负偏差：部分满足技术条件需求，其中未能满足的应明确列明。
2. 应答人声明：针对本采购标的，除本表已列明偏离外，我们接受采购文件规定的其余全部技术条件，并承诺按照采购文件规定的技术条件提供对应产品和服务。
```

## 四、偏离表编制策略

### 1. 尽量做到零偏离
- 技术偏离直接影响评分（每条负偏离扣1分）
- 商务实质性条款偏离导致废标
- 投标前应仔细评估所有条款的可满足性

### 2. 正偏差的运用
- 正偏差可展示投标人的技术优势和附加价值
- 不会扣分，但需确保所承诺的正偏差内容确实能够交付
- 适合用于展示AI能力、信创适配、额外功能模块等增值服务

### 3. 负偏差的处理
- 必须在表中明确标注负偏差项，不得隐瞒
- 应说明未满足的原因及替代方案
- 提供明确的解决时间表和承诺
- 负偏差越少越好，每一条都需慎重评估是否值得

### 4. 逐条对照
- 对照招标文件技术需求书的每一项要求逐条检查
- 确保不遗漏任何条款
- 建议使用检查表方式逐项确认

## 五、常见偏离场景参考

| 场景 | 类型 | 处理建议 |
|------|------|---------|
| 功能完全满足 | 满足 | 直接填写"满足" |
| 功能满足且有增强 | 正偏差 | 详细说明增强内容 |
| 某功能需定制开发 | 满足 | 通过定制开发满足，不算偏离 |
| 第三方接口暂不支持 | 负偏差 | 说明计划实现时间和方案 |
| 信创环境未完全适配 | 负偏差 | 说明适配计划和时间节点 |
| 提供额外AI功能 | 正偏差 | 作为增值亮点详细描述 |

> 偏离表的编制质量直接反映投标人对招标文件的理解深度和方案的完整性，应给予高度重视。
