# 技术框架与选型

## 一、整体架构概述

系统整体采用微服务架构、前后端分离开发模式，业务功能基于微服务开发平台、低代码平台以及其他配套工具进行开发，实现业务的灵活定制。系统遵循规范化、自主可控原则，系统框架技术确保先进成熟、自主可控；系统架构中各层采用成熟的、符合技术标准的国产化服务器、数据库产品。

移动APP支持iOS、Android、HarmonyOS操作系统，PC端支持兼容Chrome、Firefox、Edge、360极速浏览器等主流浏览器。

## 二、核心开发语言与框架

### 2.1 Java 技术体系

平台采用Java技术语言作为核心开发语言，这主要基于Java在企业级复杂应用开发中的成熟度、稳定性与强大的生态系统。系统后端基于JDK 1.8版本开发，设计及开发符合J2EE体系结构及规范。Java是目前主流编程语言，J2EE体系及Spring生态体系具有完善的二次开发平台，能够满足企业级投资管理信息系统的长期发展需求。

JDK选型方面，为满足信创要求，推荐采用毕昇JDK（bisheng-jdk-8u452），整个平台选型基于Java 8。

### 2.2 Spring 生态体系

后端应用服务采用基于Spring Boot、Spring Cloud、Spring Cloud Alibaba的微服务框架，实现页面和具体业务的快速开发。具体组件选型如下：

- **网关组件**：采用Spring Cloud Gateway作为网关基础组件，实现统一的路由转发、限流、熔断等操作
- **注册与配置中心**：采用Nacos 2.1.1作为注册中心和配置中心，实现服务的自动发现与配置集中管理
- **分布式事务**：使用Seata 1.8.0作为分布式事务组件，保障跨服务数据一致性
- **规则引擎**：采用开源规则引擎平台Drools 7，提供开放的封装业务规则的计算组件
- **工作流引擎**：采用自主研发的基于BPMN 2.0的工作流平台，对工作流程可动态配置，提供基于浏览器的流程设计器
- **定时任务**：采用xxl-job分布式定时任务框架，实现服务定时调用

### 2.3 前端技术栈

前端工程（包括移动端）全部采用 H5 + Vue 3 + vue-router + Pinia + Webpack 技术组成，配合 Element UI、Vant 等 UI 组件库完成前端项目的开发。前端环境主要依赖 Node.js 运行环境，依赖包使用 npm 安装，并通过 Nexus 私服仓库中建立的 npm 私服加快前端依赖下载速度。

前端的测试环境、生产环境不需要在服务器上安装 Node 环境，只需要在 Jenkins 中安装需要的 Node.js 版本即可，配合自动化部署配置实现持续集成。

### 2.4 移动端技术路线

为适配集成原生移动应用、混合应用等移动端技术，移动端采用 H5 APP 开发方式。基础技术为 H5 + CSS3 + ES6，前端搭建主要采用 Vue + vue-router + Vuex + Webpack 技术组成的前台框架，UI层使用 Element UI 和 Vant 保证美观易用高效，支持在 Android、iOS、HarmonyOS 上运行。

移动端平台包括两部分：一是基于 UniApp 的基座应用，主要考虑到其高性能，非常接近原生APP应用，用户体验方面流畅，同时打包生成各系统适配应用；二是基于 H5 的业务应用，主要用于实现各功能业务操作和展示。

## 三、微服务架构设计

### 3.1 架构分层

系统采用横向分层架构，严格划分前端层、API网关层、业务服务层、数据存储层等核心层级，层间通过标准化接口实现交互，无直接依赖关系。每层内部按功能拆分为独立模块，如业务服务层细分为客户管理、合同管理、风控校验等模块，各模块采用独立代码仓库、独立部署模式，通过容器化技术实现资源隔离。

纵向按"业务专属逻辑 + 可复用公共服务"进行分割，将客户认证、数据校验、权限控制、通知推送等通用能力抽离为独立微服务，通过服务注册中心统一管理，供各类业务调用。新增产品可通过调用可复用的服务实现自身的业务逻辑，而对现有产品没有任何影响。

### 3.2 组件化结构

系统采用全组件化结构设计，将核心功能拆解为独立封装的功能组件，涵盖业务处理、数据管理、风控校验、报表统计等全场景模块。每个组件具备完整的独立功能边界，通过"即插即用"的设计模式实现灵活配置与组合。所有功能实体间的数据交换以及对其他模块的数据引用都通过标准接口完成（RESTful API、标准化消息协议等），彻底消除组件间的直接依赖。

系统还提供低代码开发平台，支持业务功能快速配置和流程自定义，封装二次开发应用工具包，使其他技术团队对平台进行二次开发时能够更好地复用。

## 四、数据库选型

数据库采用达梦数据库DM8版本，访问频率较高的业务数据使用tongRDS分布式缓存。达梦数据库拥有完全自主知识产权，充分结合国内数据库应用的特点，在大数据量处理、高并发访问、高可用性和安全性等方面进行了深度优化。

数据库采用双机热备、数据复制的方式实现高可用部署。按照开发环境、测试环境、生产环境每个环境安装一套，保障环境隔离。

## 六、DevOps 与部署

整个开发流程融入DevOps理念，使用Docker进行容器化封装，配合CI/CD工具链实现自动化部署与交付。具体工具链如下：

- **容器技术**：选择Docker作为容器化技术，使用Kubernetes作为容器编排工具，实现自动化容器部署与复制、弹性扩缩容、负载均衡、滚动升级
- **持续集成与部署**：选用Jenkins作为持续集成和部署工具，通过自动化构建、测试和部署流程，提高开发效率
- **监控和日志**：选择Zabbix、Prometheus、Grafana作为监控工具；使用ELK作为日志管理工具，用于实时监控系统性能指标和处理异常情况
- **接口文档管理**：使用YApi可视化接口配置平台，帮助开发者创建、发布、维护API

系统包括开发、测试、生产三套环境，为系统部署搭建全套DevOps自动化管理工具，覆盖需求管理、计划管理、代码管理、持续集成、缺陷管理、自动化测试、持续部署、持续交付流水线，可视化研发管理仪表盘等功能。

## 七、技术选型理由总结

1. **Java + Spring生态**：成熟稳定，企业级开发首选，具备完善的生态系统和丰富的人才储备
2. **微服务架构**：实现服务解耦与独立部署，支持弹性伸缩，提升系统高可用能力
3. **前后端分离**：前后端独立开发、测试、部署，提高开发效率和系统灵活性
4. **国产化技术栈**：达梦数据库、TongWeb应用服务器、TongRDS缓存等国产产品，满足信创合规要求
5. **容器化部署**：Docker + Kubernetes 实现环境一致性、快速部署和弹性扩容
6. **DevOps理念**：CI/CD自动化流水线，提高交付效率和系统质量
