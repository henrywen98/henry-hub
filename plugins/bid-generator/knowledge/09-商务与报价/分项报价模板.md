# 分项报价模板

## 一、投标分项汇总表模板

以下为标准的投标分项汇总表 Markdown 格式模板，投标时需根据招标文件要求调整列字段和模块划分。

### 模板格式

```
投标分项汇总表

项目名称：{项目全称}
招标编号：{招标编号}
投标人名称：{公司全称}

| 序号 | 服务内容 | 备注（功能范围说明） | 合计（元） |
|------|---------|---------------------|-----------|
| 1    | {模块名称1} | {包含的具体功能模块简述} | ￥{金额} |
| 2    | {模块名称2} | {包含的具体功能模块简述} | ￥{金额} |
| 3    | {模块名称3} | {包含的具体功能模块简述} | ￥{金额} |
| ...  | ...     | ...                 | ...       |
|      | **合计** |                     | **￥{总额}** |

备注：
1、投标人根据实际情况自行扩展报价，合计金额须与投标书、开标一览表中的价格一致。
2、如果不提供分项报价将视为没有实质性响应招标文件。
3、如果按单价计算的结果与总价不一致，以单价为准修正总价。

投标人授权代表姓名（签字或盖章）：
投标人名称：{公司全称}（公章）
日期：{年} 年 {月} 月 {日} 日
```

## 三、常见模块报价分类

根据历史投标经验，信息化系统项目分项报价可按以下维度拆分：

### 按功能模块拆分（推荐）
适用于功能边界清晰的系统建设项目。

| 类别 | 示例模块 |
|------|---------|
| 综合办公 | 公文管理、流程审批、档案管理、会议管理 |
| 财务管理 | 预算、报销、核算、报表、资金支付 |
| 业务核心 | 投资管理、基金管理、风控管理、项目管理 |
| 人力资源 | 人员信息、招聘、绩效、薪资、培训 |
| 监督合规 | 内控评价、风险预警、审计追踪 |
| 系统基础 | 统一门户、权限管理、数据接口、移动端 |

### 按服务阶段拆分
适用于需要区分建设与运维的项目。

| 类别 | 包含内容 |
|------|---------|
| 软件开发费 | 需求分析、系统设计、编码开发、系统测试 |
| 实施部署费 | 环境搭建、数据迁移、系统集成、上线部署 |
| 培训费 | 管理员培训、业务用户培训、操作手册 |
| 维保服务费 | 免费维保期后的年度维保费用 |

## 四、开标一览表模板

```
开标一览表

项目名称：{项目全称}
招标编号：{招标编号}
投标人名称：{公司全称}

| 投标价格（人民币元） | 投标保证金 | 服务期 | 备注 |
|---------------------|-----------|--------|------|
| 小写：{金额}元      | {保证金}元 | {天数}天 | 无   |
| 大写：{大写金额}整   |           |        |      |

注：上述报价须按照投标分项汇总表中的各项规格、数量进行编制。

投标人授权代表姓名（签字或盖章）：
投标人名称：{公司全称}（公章）
日期：{年} 年 {月} 月 {日} 日
```

> 编制报价时务必确保：投标书总价 = 开标一览表价格 = 分项汇总表合计金额，三者完全一致。
